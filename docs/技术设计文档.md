# MikMok 视频平台技术设计文档

## 1. 技术栈
*   **前端**: React 18 (Vite) + Tailwind CSS + TanStack Query (数据同步与缓存)
*   **后端**: Spring Boot 3.x (Java 17+)
*   **数据库**: PostgreSQL 15
*   **缓存/排行榜**: Redis 7
*   **对象存储**: MinIO
*   **安全框架**: Spring Security + JWT
*   **部署**: Docker Compose (包含 Nginx 反向代理)

## 2. 数据库设计 (PostgreSQL)

### 2.1 用户表 `users`
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | BigSerial | 主键 |
| username | varchar(50) | 用户名 (唯一) |
| password | varchar(100) | 加密密码 |
| role | varchar(20) | 角色 (USER, ADMIN) |

| 字段名 | 类型 | 说明 |

| title | varchar(255) | 视频标题 |

### 2.3 观看历史表 `watch_history`

## 3. Redis 数据结构

### 3.1 视频热度排行榜 (ZSet)
*   **Key**: `mikmok:video:rank`
*   **Score**: `like_count` (点赞数)
*   **Member**: `video_id`
*   **用途**: 快速获取全站最火视频 ID 列表。

### 3.2 用户已看集合 (Set)
*   **Key**: `mikmok:user:watched:{user_id}`
*   **Member**: `video_id`
*   **用途**: 在推荐时快速过滤已看视频。系统启动或用户登录时可从数据库同步，新观看记录实时写入。

### 3.3 接口耗时统计 (Hash)
*   **Key**: `mikmok:stats:duration`
*   **Field**: `{path}:{method}`
*   **Value**: 平均耗时/最大耗时等统计数据（可选，用于后台快速展示）。

## 4. 核心功能实现方案

### 4.1 视频推荐逻辑 (AOP + Redis)
1.  从 Redis `mikmok:video:rank` 获取 Top 100 的视频 ID。
2.  获取当前用户的 `mikmok:user:watched:{user_id}` 集合。
3.  在 Java 内存中使用 `Set.removeAll()` 剔除已看 ID。
4.  根据分页参数（Offset, Limit）截取结果。
5.  如果结果不足，从数据库补位（兜底逻辑）。

### 4.2 监控能力 (AOP)
*   定义 `@Monitor` 注解或拦截所有 `@RestController`。
*   使用 `StopWatch` 记录执行时间。
*   通过 `ContentCachingRequestWrapper` 和 `ContentCachingResponseWrapper` 获取请求和响应的 Body。
*   **异步写入**: 日志记录通过 `ThreadPoolTaskExecutor` 异步保存到 PostgreSQL，不阻塞主业务流程。

### 4.3 视频存储 (MinIO)
*   前端上传文件到 Spring Boot 后端。
*   后端生成唯一文件名（UUID），调用 MinIO Java SDK 上传。
*   上传成功后，将 MinIO 生成的 Object Name 或完整 URL 存入数据库。

### 4.4 安全校验
*   使用 **JWT** 进行无状态认证。
*   **权限控制**:
    *   `DELETE /api/videos/{id}`: Service 层校验 `video.user_id == current_user.id`。
    *   `/api/admin/**`: 校验 `current_user.role == 'ADMIN'`。

## 5. 部署架构 (Docker Compose)
*   **frontend**: React 静态资源，由 Nginx 托管。
*   **backend**: Spring Boot 应用镜像。
*   **postgres**: 挂载 `./data/postgres` 持久化数据。
*   **redis**: 挂载 `./data/redis` 持久化数据。
*   **minio**: 挂载 `./data/minio` 持久化视频文件。
*   **nginx**: 作为入口，转发 `/api` 到 backend，其余请求指向 frontend。

## 6. 开发与部署环境

### 6.1 混合开发模式 (Hybrid Mode)
为了保证团队协作时数据的一致性，采用以下配置：
*   **云端服务器**: 运行 Docker Compose 部署 PostgreSQL, Redis, MinIO。
*   **本地开发**: 组员在各自电脑运行 Spring Boot 和 React，通过公网 IP 连接云端服务。

### 6.2 关键配置项
*   **数据库连接**: `jdbc:postgresql://<SERVER_IP>:5432/mikmok`
*   **Redis 连接**: `<SERVER_IP>:6379`
*   **MinIO Endpoint**: `http://<SERVER_IP>:9000`
*   **MinIO Console**: `http://<SERVER_IP>:9001` (用于管理员手动管理文件)

## 7. 总结
本设计通过 Redis 缓存高频读写操作，利用 AOP 实现全量接口监控，并结合 MinIO 提供稳定的视频存储能力，能够高效满足 MikMok 平台的功能与性能需求。
