# MikMok 视频平台需求文档

## 1. 项目概述
MikMok 是一个短视频分享与推荐平台，支持用户上传视频、点赞视频，并根据点赞热度为用户推荐其未看过的视频。系统分为前台展示系统和后台管理系统，具备完善的日志监控和安全校验能力。

## 2. 角色定义
*   **游客**：可以注册、登录。
*   **登录用户**：可以浏览视频、点赞、发布视频、管理自己的视频。
*   **管理员**：可以进入后台管理系统，查看系统运行监控数据。

## 3. 功能需求

### 3.1 前台功能（面向用户）

#### 3.1.1 视频推荐主页
*   **全屏滚动浏览**：采用类似抖音的 Snap Scroll 效果，用户通过鼠标滚轮或滑动进行视频切换。
*   **热度推荐**：系统根据视频点赞数从高到低进行推荐。
*   **访问去重**：用户已看过的视频不再出现在推荐流中（需持久化记录）。
*   **分页加载**：视频流采用分页获取方式（如每页 10 条），确保浏览流畅度。
*   **视频点赞**：用户可以对喜欢的视频点赞，点赞数实时更新并影响推荐排名。

#### 3.1.2 我的视频管理
*   **发布视频**：用户可以上传视频文件，填写视频标题。
*   **视频列表**：分页展示用户自己发布的视频。
*   **删除视频**：用户可以删除自己发布的视频（需权限校验）。

#### 3.1.3 用户管理
*   **注册**：新用户账号注册。
*   **登录**：用户登录验证。
*   **退出登录**：结束当前登录会话。

### 3.2 后台功能（面向管理员）

#### 3.2.1 系统监控面板
*   **请求日志查看**：以表格形式展示所有用户请求的具体信息，包括：
    *   请求路径 (URL)
    *   用户 ID
    *   输入参数 (Request Body/Params)
    *   输出结果 (Response Body)
    *   请求时间戳
*   **接口耗时监控**：记录并展示每个接口的执行耗时（单位：ms），支持查看耗时分布。

## 4. 非功能需求

### 4.1 存储设计
*   **数据库**：使用 PostgreSQL 存储用户信息、视频元数据、观看历史和系统日志。
*   **视频存储**：使用 MinIO 对象存储服务存放视频文件，数据库仅记录访问 URL。
*   **高性能缓存**：使用 Redis 处理视频点赞排行榜、用户已看记录去重以及接口耗时统计。

### 4.2 安全校验
*   **账号系统**：所有非公开接口需进行登录状态校验。
*   **权限控制**：
    *   删除视频接口必须校验当前用户是否为该视频的发布者。
    *   后台监控页面仅限管理员权限访问。

### 4.3 日志与监控
*   **全量记录**：通过 AOP 或拦截器技术，对所有 Controller 接口进行无侵入式监控。
*   **输入输出记录**：日志必须包含完整的请求和响应报文。
*   **耗时统计**：精准记录从请求进入到响应返回的完整生命周期耗时。

## 5. 核心业务逻辑

### 5.1 推荐策略 (C方案)
1.  后端根据点赞数从 Redis ZSet 获取候选视频 ID。
2.  结合数据库/Redis 中的用户已看记录进行过滤。
3.  按分页大小返回过滤后的视频详情。
4.  前端预加载下一页内容，确保滑动体验。

### 5.2 视频去重逻辑
*   用户每完整观看或点赞一个视频，系统将该视频 ID 存入 `user_watch_history` 表。
*   推荐查询时，使用 `NOT IN` 或 `EXCEPT` 逻辑排除已看 ID。
